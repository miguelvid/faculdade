1. Introdução

1.1 Propósito

Este documento especifica os requisitos de software para o Sistema de Gerenciamento de Transportes Rodoviários (SGBUS). Destina-se a ser utilizado pela equipe de desenvolvimento e stakeholders do projeto.

1.2 Escopo

O SGBUS é um sistema que permitirá o gerenciamento completo de operações de empresas de transporte rodoviário, incluindo venda de passagens, gestão de frota, controle de rotas e administração de pessoal.

1.3 Definições, Acrônimos e Abreviações

  • SGBUS: Sistema de Gerenciamento de Transportes Rodoviários
  • ANTT: Agência Nacional de Transportes Terrestres
  • SLA: Service Level Agreement (Acordo de Nível de Serviço)
  • API: Application Programming Interface
  • CRUD: Create, Read, Update, Delete

1.4 Referências

  1. IEEE Std 830-1998 - IEEE Recommended Practice for Software Requirements Specifications
  2. Regulamentação ANTT Nº 4.770/2015 - Transporte Rodoviário Interestadual
  3. Lei Geral de Proteção de Dados (LGPD) - Lei nº 13.709/2018

2. Descrição Geral

2.1 Perspectiva do Produto

O SGBUS é um sistema independente que se integrará com:

  • Sistemas de pagamento online
  • APIs de geolocalização
  • Sistemas governamentais de controle de transporte
  • Sistemas de gestão empresarial (ERP)

2.2 Funções do Produto

  • Gestão de Passagens: Venda online, Reservas, Cancelamentos, Reembolsos
  • Gestão de Frota: Cadastro de veículos, Controle de manutenção, Monitoramento em tempo real, Gestão de combustível
  • Gestão de Pessoal: Escalas de motoristas, Controle de jornada, Gestão de documentação
  • Gestão de Rotas: Planejamento de itinerários, Controle de horários, Gestão de paradas
  • Relatórios e Analytics: Relatórios operacionais, Indicadores financeiros, Análises de desempenho

2.3 Características dos Usuários

Tipo Perfil Nível Técnico Privilégios
Passageiros Usuários finais do serviço de transporte Básico Acesso limitado a funções de compra e consulta
Motoristas Profissionais de transporte Intermediário Acesso a funções operacionais específicas
Administradores Gestores do sistema Avançado Acesso total ao sistema

2.4 Restrições

  • Conformidade com regulamentações da ANTT
  • Adequação à LGPD
  • Normas de segurança digital

3. Requisitos Específicos

3.1 Requisitos de Interface Externa

3.1.1 Interfaces do Usuário

  • Interface Web (Passageiros)
    • RF1.1: Tela de busca de viagens
    • RF1.2: Sistema de compra de passagens
    • RF1.3: Área do cliente
  • Aplicativo Móvel (Motoristas)
    • RF2.1: Visualização de rotas
    • RF2.2: Registro de ocorrências
    • RF2.3: Controle de viagem
  • Sistema Administrativo
    • RF3.1: Dashboard gerencial
    • RF3.2: Gestão de usuários
    • RF3.3: Configurações do sistema

3.1.2 Interfaces de Hardware

  • Dispositivos de Rastreamento
    • RH1.1: Integração com GPS
    • RH1.2: Monitoramento de telemetria
  • Equipamentos de Venda
    • RH2.1: Impressoras térmicas
    • RH2.2: Leitores de cartão

3.1.3 Interfaces de Software

  • Integrações Externas
    • RS1.1: APIs de pagamento
    • RS1.2: Serviços de mapas
    • RS1.3: Sistemas governamentais

3.2 Requisitos Funcionais

3.2.1 Gestão de Passagens

  • RF4: O sistema deve permitir a compra de passagens online
    • RF4.1: Seleção de origem e destino
    • RF4.2: Escolha de data e horário
    • RF4.3: Seleção de assento
    • RF4.4: Processamento de pagamento

3.2.2 Gestão de Frota

  • RF5: O sistema deve gerenciar a frota de veículos
    • RF5.1: Cadastro de veículos
    • RF5.2: Controle de manutenção
    • RF5.3: Monitoramento em tempo real

3.3 Requisitos Não Funcionais

3.3.1 Desempenho

  • RNF1: Tempo de resposta máximo de 3 segundos
  • RNF2: Suporte a 5.000 usuários simultâneos
  • RNF3: Disponibilidade de 99,9%

3.3.2 Segurança

  • RNF4: Criptografia de dados sensíveis
  • RNF5: Autenticação em duas etapas
  • RNF6: Registro de todas as operações
